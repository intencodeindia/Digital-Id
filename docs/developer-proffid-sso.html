Proffid SSO Integration

What is Proffid SSO?

Proffid SSO is a secure authentication system that allows users to access multiple applications with a single set of credentials.

OAuth 2.0 & OpenID Connect

Quick Start

1. Get Your API Credentials
Your API Endpoints:
Authorization: https://auth.proffid.com/oauth/authorize
Token: https://auth.proffid.com/oauth/token
UserInfo: https://auth.proffid.com/oauth/userinfo
2. Basic Implementation

// Initialize the OAuth client
$config = [
    'client_id' => 'your_client_id',
    'client_secret' => 'your_client_secret',
    'redirect_uri' => 'https://your-app.com/callback',
    'auth_endpoint' => 'https://auth.proffid.com/oauth/authorize',
    'token_endpoint' => 'https://auth.proffid.com/oauth/token',
    'userinfo_endpoint' => 'https://auth.proffid.com/oauth/userinfo'
];

// Redirect to authorization page
public function redirectToSSO()
{
    $params = [
        'client_id' => $config['client_id'],
        'redirect_uri' => $config['redirect_uri'],
        'response_type' => 'code',
        'scope' => 'email profile',
        'state' => csrf_token()
    ];
    
    return redirect($config['auth_endpoint'] . '?' . http_build_query($params));
}

Security Best Practices

Always use HTTPS

Ensure all OAuth endpoints and redirect URIs are served over HTTPS.

Validate the state parameter to prevent CSRF attacks.

Store tokens securely in HTTP-only cookies or encrypted in the database.

Implement proper token expiration handling.

FAQ

How do I get my API credentials?

Contact the Proffid admin team to register your application and obtain your client ID and secret.

What OAuth 2.0 flows are supported?

Currently, Proffid SSO supports the Authorization Code flow, which is recommended for server-side applications.

How do I handle token expiration?

Implement token refresh logic using the refresh token endpoint. Always check token expiration before making API calls.

What scopes are available?

Basic scopes include 'email' and 'profile'. Contact support for additional custom scopes for your application.

Is HTTPS required?

Yes, HTTPS is mandatory for all OAuth endpoints and redirect URIs to ensure secure communication.

What user data is available through the API?

The userinfo endpoint provides basic profile data including email, name, and profile picture. Additional fields may be available based on granted scopes.

How do I report security issues?

Email security@proffid.com immediately for any security concerns. Include detailed information about the vulnerability.
